1. O que é um Namespace?
Intenção: Um Namespace (Espaço de Nomes) funciona como um "sobrenome" ou uma "caixa" que agrupa comandos, classes e funções que são logicamente relacionados, evitando conflitos de nomes.

Problema Resolvido: Se você criar uma função chamada cout no seu código, e tentar usar o cout padrão do C++, o compilador ficaria confuso. O Namespace std resolve isso.

2. O Namespace std (Padrão)
Rigidez: O C++ armazena todas as suas funcionalidades básicas e padrão (como cout, cin, endl, vector, string, etc.) dentro do Namespace std (Standard).

Acesso Explícito (A Melhor Prática):

Você acessa um comando dentro do std usando o operador de escopo ::.

Sintaxe: std::cout, std::cin.

Vantagem: É a forma mais segura e rigorosa, pois evita o problema de conflito de nomes (name collision). É o método recomendado para código profissional.

3. Simplificando com a Diretiva using
A diretiva using oferece atalhos para comandos, mas deve ser usada com cautela:

A. Abertura Total (Não Recomendado)
C++

using namespace std;
Intenção: Torna todos os comandos dentro do Namespace std disponíveis diretamente, sem precisar escrever std:: (ex: você pode usar apenas cout em vez de std::cout).

Desvantagem (Rigidez): Polui todo o escopo do arquivo. Se você tiver uma função com o mesmo nome de uma função em std, haverá um erro de ambiguidade. Deve ser evitado em arquivos de cabeçalho (.h) ou projetos grandes.

B. Abertura Seletiva (Uso Aceitável)
C++

using std::cout;
using std::endl;
Intenção: Traz apenas os comandos específicos que você deseja usar com frequência para o escopo atual (ex: cout, endl).

Vantagem: Reduz a digitação, mas minimiza o risco de conflito de nomes, pois apenas alguns nomes são importados.
